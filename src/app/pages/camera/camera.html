技術仕様メモ

各ボタンは HTML の <input type="file"> に以下の属性を付けて実現しています:

  • accept="image/*" — 画像のみを受け取る
  • capture="environment" — 背面カメラを優先起動 (ヒント指定)
  • 属性なし — 通常のファイル選択 (ライブラリから選ぶ)

※ 自撮り側 (フロント / capture="user") ボタンは業務ユースケースに不要なため削除しています。

iOS Safari / Android Chrome ともに capture 属性に対応しており、OS ごとに実装を分岐する必要はありません。

既知の挙動差:

  • capture 属性は「ヒント」であり、カメラアプリが前/後を区別する保証はありません (多くの純正カメラは区別しない)
  • アプリ内ブラウザ (LINE / X / Facebook 等) は capture 属性を無視してライブラリを開くことが多いです
  • 確実にフロント/リアを切り替えたい場合は navigator.mediaDevices.getUserMedia() でカスタムカメラ UI を実装する必要があります
